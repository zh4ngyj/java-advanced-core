# 设计模式: 适配器模式 (Adapter)

适配器模式是一种结构型模式，它允许将一个类的接口转换成客户端期望的另一个接口。适配器模式让那些由于接口不兼容而不能在一起工作的类可以协同工作。

## 核心思想
就像一个电源适配器，它使得不同标准的插头可以在不兼容的插座上使用。在软件中，适配器扮演着中间转换的角色，解决两个已有接口之间的不兼容问题。

## 角色
1.  **目标接口 (Target)**: 客户端期望使用的接口。
2.  **被适配者 (Adaptee)**: 需要被适配的、接口不兼容的类。
3.  **适配器 (Adapter)**: 实现目标接口，并持有一个被适配者对象的引用。它将客户端对目标接口的调用转换为对被适配者接口的调用。

## 实现方式

### 1. 类适配器 (Class Adapter)
- **实现方式**: 基于**继承**。适配器类同时继承被适配者类并实现目标接口。
- **缺点**: 由于 Java 的单继承限制，这种方式不够灵活。适配器只能适配一个被适配者类。

### 2. 对象适配器 (Object Adapter)
- **实现方式**: 基于**组合**。适配器类实现目标接口，并在内部持有一个被适配者对象的实例。
- **优点**:
    - **灵活性高**: 可以适配被适配者类及其所有子类。
    - **推荐使用**: 这是更常用、更灵活的实现方式。

## 与其他模式的区别

- **与装饰器模式 (Decorator) 的区别**:
    - **意图**: 适配器旨在**统一接口**，而装饰器旨在**增强功能**。
    - **实现**: 装饰器通常与被装饰者实现相同的接口，而适配器则用于连接两个不同的接口。

- **与代理模式 (Proxy) 的区别**:
    - **意图**: 代理模式控制对原始对象的访问，提供与原始对象相同的接口。适配器模式则是改变接口。

## 经典应用
- **`java.io` 中的 `InputStreamReader` 和 `OutputStreamWriter`**:
    - `InputStreamReader` 是一个适配器，它将字节流 (`InputStream`) 适配成字符流 (`Reader`)。
    - `OutputStreamWriter` 将字符流 (`Writer`) 适配成字节流 (`OutputStream`)。
- **`java.util.Arrays.asList()`**: 该方法将一个数组适配成一个 `List` 对象。
