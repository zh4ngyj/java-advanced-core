# 模块: Java 核心关键字

本模块深入探讨 Java 中几个至关重要的关键字，理解它们的底层原理和使用场景是高级开发的必备技能。

## 1. `static`
- **定义**: `static` 关键字用于声明属于类本身而不是类的实例的成员（变量或方法）。
- **核心特性**:
    - **类级别**: 静态成员在类加载时被初始化，早于任何对象的创建。
    - **共享性**: 所有类的实例共享同一份静态成员。任何实例修改了静态变量，其他实例看到的值都会改变。
    - **访问**: 可以通过类名直接访问 (`ClassName.staticMember`)，也可以通过对象引用访问（不推荐）。
- **使用场景**:
    - **工具类**: `Math` 类中的所有方法都是静态的。
    - **共享常量**: `public static final` 定义全局常量。
    - **单例模式**: 用于持有唯一的实例。

## 2. `final`
- **定义**: `final` 用于声明一个实体（变量、方法或类）是最终的，不可改变的。
- **核心特性**:
    - **修饰变量**:
        - **基本类型**: 值不可改变。
        - **引用类型**: 引用的地址不可改变，但引用指向的对象内容是可变的。
    - **修饰方法**: 方法不可被子类重写 (override)。
    - **修饰类**: 类不可被继承。例如 `String` 类就是 `final` 类。

## 3. `volatile`
- **定义**: `volatile` 是一个变量修饰符，它确保了在多线程环境中对该变量的修改具有**可见性**，并**禁止指令重排序**。
- **核心特性**:
    1.  **可见性 (Visibility)**: 当一个线程修改了 `volatile` 变量的值，这个新值对其他线程是立即可见的。它通过强制从主内存读取，而不是使用线程本地缓存来实现。
    2.  **有序性 (Ordering)**: 禁止编译器和处理器对 `volatile` 变量相关的读写操作进行指令重排序，从而避免了在特定并发场景下（如DCL单例模式）的逻辑错误。
- **与 `synchronized` 的区别**:
    - `volatile` 是轻量级的同步机制，只保证可见性和有序性，不保证**原子性**。
    - `synchronized` 是重量级锁，保证了可见性、有序性和原子性。

## 4. `transient`
- **定义**: `transient` 关键字用于在对象序列化时，标记某些成员变量不参与序列化过程。
- **核心特性**:
    - **序列化排除**: 当一个对象被序列化时（例如写入文件或通过网络传输），被 `transient` 修饰的变量值会被忽略，反序列化时该变量会被赋予其类型的默认值（如 `null` 或 `0`）。
- **使用场景**:
    - **敏感数据**: 密码、密钥等不应被持久化或传输的字段。
    - **可派生数据**: 可以从其他字段计算得出的变量，无需序列化以减小体积。
    - **不可序列化的字段**: 当一个类实现了 `Serializable` 接口，但其某个成员变量的类型没有实现 `Serializable` 时，需要用 `transient` 标记。
