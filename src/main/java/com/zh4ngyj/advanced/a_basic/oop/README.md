# 模块: 面向对象编程 (OOP)

这个包专注于演示和解释Java面向对象编程的核心思想。

## 核心概念

### 1. 封装 (Encapsulation)
- **定义**: 将数据（属性）和操作数据的方法（行为）捆绑到一个单元（类）中，并对外部隐藏对象的内部状态。外部只能通过对象暴露的公共接口（public methods）来访问数据。
- **目的**:
    - **信息隐藏**: 保护对象内部数据不被外部随意篡改，保证数据的完整性和安全性。
    - **降低耦合**: 内部实现可以自由修改，只要公共接口不变，就不会影响到外部调用方。
- **实现**: 使用访问修饰符 (`private`, `protected`, `public`)。通常将属性设为 `private`，并提供 `public` 的 `getter/setter` 方法。

---

### 2. 继承 (Inheritance)
- **定义**: 一个类（子类/派生类）可以继承另一个类（父类/基类）的属性和方法。子类可以重用父类的代码，并可以添加自己的新特性或重写（override）父类的方法。
- **目的**:
    - **代码复用**: 避免重复代码。
    - **建立层次关系**: 形成 "is-a" 的关系，符合现实世界的模型（例如：`Dog` is an `Animal`）。
- **关键字**: `extends`
- **注意事项**:
    - Java只支持单继承，一个类只能有一个直接父类，但可以通过接口实现多重继承的效果。
    - **组合优于继承 (Composition over Inheritance)**: 在很多场景下，使用组合（一个类持有另一个类对象的引用）比继承更灵活，耦合度更低。继承是强耦合（白盒复用），而组合是弱耦合（黑盒复用）。

---

### 3. 多态 (Polymorphism)
- **定义**: "一个接口，多种实现"。允许不同类的对象对同一消息做出不同的响应。多态分为编译时多态（重载）和运行时多态（重写）。
- **目的**: 提高代码的灵活性和可扩展性。通过面向接口/抽象编程，调用者无需关心对象的具体类型。
- **实现方式**:
    - **重写 (Override)**: 子类重新实现父类的方法。这是运行时多态的基础。
    - **重载 (Overload)**: 在同一个类中，方法名相同但参数列表（类型、数量、顺序）不同。
- **运行时多态的三个条件**:
    1. **继承**: 必须有继承关系。
    2. **重写**: 子类必须重写父类的方法。
    3. **父类引用指向子类对象**: 例如 `Animal animal = new Dog();`。

---
### 4. 抽象 (Abstraction)
- **定义**: 提取事物的共性，忽略其非本质的细节。在Java中，通过抽象类（`abstract class`）和接口（`interface`）来实现。
- **抽象类 vs. 接口**:
    - **抽象类**: 可以包含普通方法、抽象方法、构造函数和成员变量。用于表示"is-a"关系，并且希望代码复用。
    - **接口**: 只能包含抽象方法（Java 8后可以有默认方法和静态方法）和常量。用于表示"has-a"或"can-do"（能力）关系，强制实现者遵循契约。

---

### SOLID 原则
作为高级开发，理解和应用SOLID原则是编写高质量OOP代码的基础：
- **S - 单一职责原则 (Single Responsibility Principle)**
- **O - 开闭原则 (Open/Closed Principle)**
- **L - 里氏替换原则 (Liskov Substitution Principle)**
- **I - 接口隔离原则 (Interface Segregation Principle)**
- **D - 依赖倒置原则 (Dependency Inversion Principle)**

本包中的示例代码 (`PolymorphismDemo.java`) 旨在直观地展示这些概念。
