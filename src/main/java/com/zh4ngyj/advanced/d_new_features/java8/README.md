# 模块: Java 8 新特性

Java 8 是自 Java 5 以来最重要的版本更新，引入了大量新特性，极大地提升了 Java 的表达能力和开发效率。

## 1. Lambda 表达式
- **定义**: 一种简洁的、可以传递的匿名函数。它允许将函数作为方法的参数，或者把代码看作数据。
- **语法**: `(parameters) -> expression` 或 `(parameters) -> { statements; }`
- **核心**: Lambda 表达式的实现依赖于**函数式接口 (Functional Interface)**，即只包含一个抽象方法的接口（可以用 `@FunctionalInterface` 注解来强制约束）。
- **应用**: 大大简化了匿名内部类的写法，尤其是在事件监听、线程创建和集合操作中。

## 2. Stream API
- **定义**: 一个来自数据源的元素队列，支持聚合操作。Stream 不是数据结构，它不存储数据，而是对数据源进行计算。
- **核心特性**:
    - **链式操作**: 支持流式（fluent）的链式调用。
    - **内部迭代**: 与 `for` 循环的外部迭代不同，Stream API 使用内部迭代，将迭代过程封装起来。
    - **惰性求值**: 中间操作（如 `filter`, `map`）不会立即执行，只有当终端操作（如 `collect`, `forEach`）被调用时才会触发整个计算过程。
- **并行流 (`parallelStream`)**:
    - 可以轻松地将流操作并行化，利用多核处理器来提高性能。
    - 底层基于 `Fork/Join` 框架实现。
    - **注意**: 并行流并不总是更快，对于小数据量或CPU密集型任务，线程切换的开销可能超过并行带来的好处。同时需要注意线程安全问题。

## 3. `Optional<T>`
- **定义**: 一个可以为 `null` 的容器对象。它代表一个值存在或不存在。
- **目的**: 旨在以一种更优雅、更明确的方式处理 `null` 值，避免 `NullPointerException`。
- **常用方法**: `of()`, `ofNullable()`, `isPresent()`, `ifPresent()`, `orElse()`, `orElseGet()`, `orElseThrow()`。

## 4. 接口的默认方法和静态方法
- **默认方法 (`default` a.k.a. Defender Methods)**: 允许在接口中添加具有默认实现的方法，而不会破坏已有的实现类。主要用于接口的平滑演进。
- **静态方法 (`static`)**: 允许在接口中定义静态方法，通常用于工具类方法。

## 5. 新的日期时间 API (`java.time`)
- **背景**: 用于取代老旧、设计不良且线程不安全的 `Date` 和 `Calendar` API。
- **核心类**:
    - `LocalDate`: 表示日期 (年-月-日)。
    - `LocalTime`: 表示时间 (时-分-秒)。
    - `LocalDateTime`: 表示日期和时间。
    - `ZonedDateTime`: 带时区的日期和时间。
    - `Instant`: 表示时间戳，以 Unix 元年 (1970-01-01T00:00:00Z) 为起点。
- **优点**: 不可变、线程安全、设计清晰。
